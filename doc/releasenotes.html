Release Notes for Vaadin TestBench ${project.version}

Overview of Vaadin TestBench

Vaadin TestBench is a UI browser automation tool used for automated testing of Vaadin UIs in the browser. Vaadin TestBench is based on Selenium 2 and provides a WebDriver based API for controlling different browsers on multiple platforms.

Changes in 4.2.0

  • Screen comparison is supported for individual elements through TestBenchElement.compareScreen
  • TestBenchElement can be compared with WebElement using equals()
  • TestBenchElement has getClassNames() and hasClassName(String) for testing class names
  • The last matching element can be retrieved using ElementQuery.last()
  • Removed Vaadin version detection which was broken in Liferay
  • List selects in multiselect mode can be tested with PhantomJS
  • waitForVaadin is executed before each element method, eliminating timing issues in some tests
  • waitForVaadin handles a possible null return value
  • The hub host name can be overridden using -Dcom.vaadin.testbench.Parameters.hubHostname
  • The browser name and version to run on locally can be overridden using -Dcom.vaadin.testbench.Parameters.runLocally
  • Component Element API changes in vaadin-testbench-api 7.7.4:
    • The input field for a ComboBox can be retrieved using ComboBoxElement.getInputField()
    • GridElement has getRowCount() to get the number of data rows in the grid
    • GridElement has getRows() for iterating over all data rows in the grid
    • State of a checkbox can be checked using CheckBoxElement.isChecked()
    • Links can be clicked using LinkElement.click()
    • Link caption is correctly returned using LinkElement.getCaption()

For a complete list of changes in this release, please see the of closed issues.

Package Contents

The package contains the following files and directories: 

documentation/
 The documentation folder contains a PDF excerpt of the TestBench chapter in Book of Vaadin, and information about the license. 
maven/
 The Maven folder contains a version of the Vaadin TestBench libraries that you can install in your local Maven repository along with short instructions on how to install. Use these if you cannot access the Vaadin add-ons maven repository at http://maven.vaadin.com/vaadin-addons. 
vaadin-testbench-standalone-${project.version}.jar
 This is the standalone version of the Vaadin TestBench library. It includes the Selenium 2 WebDriver along with all other dependencies and is suitable for running the grid hub and nodes. See the manual for details.

Third Party Dependencies

All dependencies, along with their dependencies are bundled in vaadin-testbench-standalone-${project.version}.jar. The maven directory also contains vaadin-testbench-${project.version}.jar, which does not bundle any dependencies. Vaadin TestBench 4 directly depends on the following third-party libraries:

  • Selenium 2.51.0
  • PhantomJSDriver 1.2.1
  • Javassist 3.18.1-GA
  • JUnit 4.12
  • org.json

Migration Notes

Vaadin TestBench 4 is API compatible with Vaadin TestBench 3.x, and can thus be used as a drop-in replacement for any 3.x version. The exceptions to this rule are listed in the API changes section below.

API Changes

  • closeNotification() has been removed from TestBenchElement and is now NotificationElement.close()

Supported Platforms and Browsers

Vaadin TestBench 4 fully supports Windows, Mac OS X and Linux as well as iOS and Android for running tests. The following browsers are supported:

  • Latest Google Chrome
  • Internet Explorer 8, 9, 10, 11
  • Latest Microsoft Edge
  • Mozilla Firefox 47 and earlier
  • Latest Safari
  • Mobile browsers:
    • Latest Android default browser
    • Latest iOS Safari
